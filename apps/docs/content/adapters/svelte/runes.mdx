`@stainless-code/svelte-layers` (package root) binds stacks to Svelte 5 runes via `createSubscriber`. The core is re-exported.

:::tip
Svelte builds compiler-free (tsdown, no `.svelte` components shipped). Render with `{#each useStack().current as s}` + `stack.callFor(s)` in your own markup â€” **intentional primitive design**, not a parity gap. See the [parity matrix](/adapters).
:::

## Install

```package-install
npm i @stainless-code/svelte-layers
```

**Peer:** `svelte` >=3.0.0 (runes entry requires **Svelte 5.7+** for `createSubscriber`)

For Svelte 3â€“4 or store-based projects, use [Svelte (store)](/adapters/svelte/store).

## Client

Call `setLayerClient()` in a parent component (creates a `LayerClient` when omitted). Descendants read it with `useLayerClient()`:

```svelte
<script>
  import { setLayerClient } from "@stainless-code/svelte-layers";

  setLayerClient();
</script>
```

:::note
Import client/stack APIs from **one** entry only. Mixing this package with `@stainless-code/svelte-layers/store` in the same tree breaks context (separate client Symbols per entry).
:::

## Render a stack

`useStack()` returns a `SvelteStack` with `.current` (the selected snapshot) and `.callFor(state)` (the call context). No `StackOutlet` â€” iterate in your markup:

```svelte
<script>
  import { useStack } from "@stainless-code/svelte-layers";

  const stack = useStack("confirm");
</script>

{#each stack.current as state (state.id)}
  {@const call = stack.callFor(state)}
  {#if call}
    <Confirm {call} payload={state.payload} />
  {/if}
{/each}
```

`useStack().current` + `callFor` is the headless equivalent of `useStackHandles` â€” no separate export.

## Confirm dialog

```svelte
<script lang="ts">
  import {
    layerOptions,
    setLayerClient,
    useLayerClient,
    useStack,
  } from "@stainless-code/svelte-layers";

  type ConfirmPayload = { title: string };
  type ConfirmResponse = boolean;

  setLayerClient();
  const client = useLayerClient();
  const confirmStack = useStack("confirm");

  const confirm = layerOptions<ConfirmPayload, ConfirmResponse>({
    stack: "confirm",
    key: ["confirm", "remove"],
  });

  async function remove() {
    await client.open({ ...confirm, payload: { title: "Remove item?" } });
  }
</script>

{#each confirmStack.current as state (state.id)}
  {@const call = confirmStack.callFor(state)}
  {#if call}
    <div role="dialog">
      <h2>{state.payload.title}</h2>
      <button onclick={() => void call.end(true)}>Yes</button>
      <button onclick={() => void call.end(false)}>No</button>
    </div>
  {/if}
{/each}

<button onclick={remove}>Remove</button>
```

## Primitives

| Export | Role |
| ------ | ---- |
| `setLayerClient(client?)` | Provide a `LayerClient` via Svelte context |
| `useLayerClient()` | Read the nearest context client |
| `useStack(stackId?, selector?, compare?)` | `SvelteStack` with `.current` and `.callFor(state)` |
| `useLayer(key, stackId?, compare?)` | `SvelteStack` whose `.current` is one `LayerState \| null` |

Both accept an optional leading `LayerClient` argument.

## Ergonomic APIs

| Export | Role |
| ------ | ---- |
| `useMutationFlow(call)` | `pending` getter (reactive via `createSubscriber`); `run(fn).orEnd(response)` |
| `useLayerGroup(call, options?)` | Child stack â€” render `{#each group.stack.current}` + `group.stack.callFor(s)` |

No `StackOutlet`, `StackSubscribe`, `useStackHandles`, or `createStackHook` â€” by design.
