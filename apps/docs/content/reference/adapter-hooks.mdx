Every adapter re-exports `@stainless-code/layers` and exposes the same primitives: client context, `useLayerClient`, `useStack`, and `useLayer`. Ergonomic wrappers (`StackOutlet`, `useLayerGroup`, `useMutationFlow`, `createStackHook`) sit on top — see the matrix below.

:::note[Svelte entries]
`@stainless-code/svelte-layers` (runes, 5.7+) and `@stainless-code/svelte-layers/store` (readable stores, 3+) share client helpers but differ in stack reactivity and rendering.
:::

## Provider / client acquisition

| Adapter | Provider / setup | Client hook |
| ------- | ---------------- | ----------- |
| React | `StackProvider({ client?, children })` | `useLayerClient(): LayerClient` |
| Preact | `StackProvider({ client?, children })` | `useLayerClient(): LayerClient` |
| Solid | `<LayerClientContext.Provider value={client}>` | `useLayerClient(): LayerClient` |
| Angular | `provideLayerClient(client?): FactoryProvider` + `LAYER_CLIENT` token | `useLayerClient(): LayerClient` (injection context) |
| Vue | `provideLayerClient(client?): LayerClient` | `useLayerClient(): LayerClient` |
| Svelte (runes) | `setLayerClient(client?): LayerClient` | `useLayerClient(): LayerClient` |
| Svelte (store) | `setLayerClient(client?): LayerClient` | `useLayerClient(): LayerClient` |

## Hook signature matrix

Return shapes per adapter — full signatures are in [Signature sketches](#signature-sketches) below. Provider/client setup is in the table above.

| Hook | React / Preact | Solid | Angular | Vue | Svelte (runes) | Svelte (store) |
| ---- | -------------- | ----- | ------- | --- | -------------- | -------------- |
| `useLayerClient` | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| `useStack` | `T` | `Accessor<T>` | `Signal<T>` | `Ref<T>` | `SvelteStack<T>` | `Readable<T>` |
| `useLayer` | `LayerState \| null` | `Accessor<…>` | `Signal<…>` | `Ref<…>` | `SvelteStack<…>` | `Readable<…>` |
| Stack render | `StackOutlet` | `StackOutlet` | `renderStack(vcr)` | `StackOutlet` | `{#each …current}` + `callFor` | `{#each $stack}` + `callFor` |
| `useLayerGroup` | `Outlet` | `Outlet` (accessors) | `renderInto(vcr)` | `Outlet` (refs) | `stack` | `stack` |
| `useMutationFlow` | `pending` | `Accessor<boolean>` | `Signal<boolean>` | `Ref<boolean>` | `boolean` getter | `Readable<boolean>` |
| `createStackHook` | `AppHost` / `AppLayer` | same | `renderInto` | same | — | — |

Solid, Angular, and Vue `useStack` / `useLayer` accept an optional leading `client: LayerClient` overload. Svelte runes/store entries follow the same pattern.

## Signature sketches

### useStack (React / Preact)

```ts
function useStack<T = LayerState[]>(
  stackId?: string,
  selector?: (states: LayerState[]) => T,
  compare?: (a: T, b: T) => boolean,
): T;
```

### Stack rendering divergences

**Angular** — no shipped `StackOutlet`. Imperative rendering into a `ViewContainerRef`:

```ts
function renderStack(
  vcr: ViewContainerRef,
  stackId?: string,
  rootProps?: unknown,
): void;
```

**Svelte** — no `.svelte` host components. Render in your markup:

```svelte
{#each stack.current as state (state.id)}
  {@const call = stack.callFor(state)}
{/each}
```

Store entry: `{#each $stack as state}` with `callFor(client, stackId, state)`.

### createStackHook

| Adapter | Returns |
| ------- | ------- |
| React, Preact, Solid, Vue | `{ StackProvider, useAppStack, AppHost, AppLayer }` |
| Angular | `{ provideClient, useAppStack, renderInto(vcr, rootProps?) }` |
| Svelte | not shipped |

:::tip[Cross-adapter porting]
Primitives + core re-export mean anything one adapter can do is reachable in another. Angular and Svelte diverge at the **rendering surface** by design (compiler-free builds).
:::
