Layers is **pre-1.0**. Breaking API changes are expected and ship in minor releases (e.g. `0.1` → `0.2`), not majors. Treat semver as a signal, not a guarantee of zero breaks until 1.0 — see [Stability & versioning](/concepts/stability).

## How breaks are announced

- Each intentional break gets a [Changelog](/changelog) entry tied to a GitHub Release.
- Migration notes will appear on this page when a release requires consumer action.

## Current status

**No breaking-change migration notes have shipped yet.** When the first semver major with consumer-facing breaks lands, step-by-step upgrade guidance will be added here.

## How migration notes will look

When a release requires consumer action, a before/after diff will appear here. Illustrative example (not a real change):
