## Seam notes

- Don't use Base UI's trigger wrappers — render the controlled overlay directly while mounted.
- `Toast.createToastManager()` owns the queue/viewport — lib owns queue; Layers is the typed `open` entry. Don't dual-queue.
- Toast is instant-dismiss (no exit-animation crux) — `call.dismiss()` after the timeout is fine; the `transition`/`call.settle()` axis is irrelevant here (it matters for animated-enter-exit, not toast).
- Layers coordinates open/dismiss only — leave the toast region to Base UI; don't add modal-style focus traps or scroll-lock.
