A drawer that opens a sub-dialog needs its own stack — but tied to the parent layer's lifetime. Layer groups create a child stack on the same `LayerClient`; when the parent dismisses, children drain automatically.

## Core primitive

`createLayerGroup(client, call, options?)` registers a child stack derived from the parent's `stackId` and instance `layerId`. Adapters wrap it as `useLayerGroup(call)`:

```ts lineNumbers
import { createLayerGroup } from "@stainless-code/layers";

// headless / custom adapter
const group = createLayerGroup(client, call);void group.open({ ...nestedConfirm, payload });
group.dismissAll();
group.dispose(); // usually automatic via adapter lifecycle
```

## Inline child outlet

Render the child stack inside the parent layer's DOM. React, Preact, Solid, and Vue ship an `Outlet` component; Lit returns `outlet(): TemplateResult`; Angular uses `renderInto`; Svelte exposes the child `stack` directly:

<Tab title="React" icon="/icons/react.svg">

```tsx lineNumbers
import { useLayerGroup } from "@stainless-code/react-layers";

function Drawer({ call }: LayerComponentProps<void, void>) {

  return (
    <div role="dialog">

import { LitElement, html } from "lit";
import { useLayerGroup } from "@stainless-code/lit-layers";
import type { LayerComponentProps } from "@stainless-code/lit-layers";

class Drawer extends LitElement {
  declare call: LayerComponentProps<void, void>["call"];
  #group?: ReturnType<typeof useLayerGroup>;

  createRenderRoot() {
    return this;
  }

  render() {
    this.#group ??= useLayerGroup(this, this.call);
    return html`<div role="dialog">
      <button
        type="button"
        @click=${() => void this.#group!.open({ ...nestedConfirm, payload })}
      >
        Edit details
      </button>
      ${this.#group.outlet()}
    </div>`;
  }
}
```

## What you get

| Member | Role |
| ------ | ---- |
| `open` | Opens on the child stack (stack id pre-bound) |
| `dismissAll` | Closes all child layers |
| `states` / `stack` | Reactive snapshot of child layers |
| `Outlet` / `outlet()` / `renderInto` / `stack` | Render surface (adapter-specific) |
| `stackId` | Derived id — `` `${parentStackId}~${parentLayerId}~${name}` `` |

:::warning[Heads up]
Child stacks cascade on parent dismiss — blockers on the parent are the right place to guard "close drawer with open sub-dialog?". Children are force-drained; see [Dismissal blockers](/guides/dismissal-blockers).
:::

No second `LayerClient`, no manual cleanup. See [Lifecycle](/concepts/lifecycle) for how dismissal propagates.
